# 创建 Pull Request 指南

## 步骤

1. 访问：https://github.com/yaox2689-max/catRAG/pull/new/develop

2. 填写以下信息：

**标题：**
```
docs: restructure README following high-star repo conventions
```

**描述：**
```markdown
## Summary

重写 README，参考 RAGFlow (82k stars)、LightRAG (36k stars)、GraphRAG (33k stars) 等高star RAG仓库的结构模式。

## Changes

### README.md 重写
- 居中标题 + 社交徽章 + 快速导航
- 精简特性表格 + 架构图
- 简化快速开始步骤
- 清晰的 API 参考和项目结构
- 贡献指南和路线图

### 新增文件
- CONTRIBUTING.md: 贡献指南
- LICENSE: MIT 许可证
- .github/ISSUE_TEMPLATE/: Issue 模板
- .github/PULL_REQUEST_TEMPLATE/: PR 模板

## Test plan

- [x] README 格式正确
- [x] 链接有效
- [x] 徽章显示正常
- [x] 项目结构清晰

## Why this matters

对于智能体开发和 AI 应用开发岗位，展示专业的工程实践能力很重要。这个 PR 体现了：

1. **规范意识**：参考高star仓库的最佳实践
2. **文档能力**：清晰、简洁、专业的项目文档
3. **工程思维**：模板化的工作流程
4. **技术深度**：RAG 系统架构和实现
```

3. 点击 "Create pull request"

## 完成后

PR 创建成功后，你可以：
- 分享 PR 链接给面试官
- 展示你的工程实践能力
- 讲解你的技术选型和设计决策
